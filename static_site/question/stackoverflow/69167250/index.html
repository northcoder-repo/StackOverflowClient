You can track the original position of the age column using data in the settings object - specifically the _ColReorder_iOrigCol value which is stored in the settings.aoColumns array of objects for each column. You can compare this value to the current column index to determine the index you need to use in the filter:

var newIdx = settings.aoColumns.filter(column => column._ColReorder_iOrigCol === 3)[0].idx;

This looks for the "orginal" column index value of 3 and returns the "current" index of that column.

You can use the above as follows:

$(document).ready(function() {

/* Custom filtering function which will search data in column four between two values */
$.fn.dataTable.ext.search.push(
    function( settings, data, dataIndex ) {
        var newIdx = settings.aoColumns.filter(column => column._ColReorder_iOrigCol === 3)[0].idx;

        var min = parseInt( $('#min').val(), 10 );
        var max = parseInt( $('#max').val(), 10 );
        var age = parseFloat( data[newIdx] ) || 0; // use data for the age column
 
        if ( ( isNaN( min ) && isNaN( max ) ) ||
             ( isNaN( min ) && age <= max ) ||
             ( min <= age   && isNaN( max ) ) ||
             ( min <= age   && age <= max ) )
        {
            return true;
        }
        return false;
    }
);
 

    var table = $('#example').DataTable({
        colReorder: true
    });
     
    // Event listener to the two range filtering inputs to redraw on input
    $('#min, #max').keyup( function() {
        table.draw();
    } );
} );

After adding the above index locator, data[3] needs to be changed to data[newIdx].

One disadvantage to this is that it will re-calculate the "new" location of the age column for every row, so there is unnecessary work being performed, using this approach.

If this is a performance problem, I would be surprised. But in that case you can consider an alternative, which is to use the column-reorder event to track changes to column ordering. You can see an example here.

I have not used that approach for your specific situation, but it should work, with some adjustments.
