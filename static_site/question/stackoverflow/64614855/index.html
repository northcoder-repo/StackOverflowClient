Using this:

div.dataTables_filter > label > input {
  font-family: Arial, sans-serif;
  font-size: .6em;
}

...works for me.

Using a css selector like .dataTables_filter > input will not work. It assumes that the input is a direct child (>) of the dataTables_filter div - but it isn't, because there is an intervening <label> element:

You should be able to avoid needing to use !important, also, this way.

"div" didn;t work for me but .dataTables_wrapper .dataTables_info, .dataTables_wrapper .dataTables_filter > label, .dataTables_wrapper .dataTables_filter > label > input worked for info and input but not for label itself (the "Search" text) - NoBullMan 3 Nov 2020 at 00:16
