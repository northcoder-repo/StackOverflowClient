A couple of observations:

  1. Handling the Namespace

  2. Getters, Setters, and Field Names

Handling the Namespace

Instead of declaring the namespace on the root element of the ListBucketResult class like this:

@XmlRootElement(name = "ListBucketResult", namespace = "http://s3.amazonaws.com/doc/2006-03-01/")

Use this instead:

@XmlRootElement(name = "ListBucketResult")

And for the namespace, handle that in a package-info.java file. This is created in your parseXML package, in the same way as your classes, but has the following contents:

@XmlSchema(
        namespace = "http://s3.amazonaws.com/doc/2006-03-01/",
        elementFormDefault = XmlNsForm.QUALIFIED)
package parseXML;

import javax.xml.bind.annotation.XmlSchema;
import javax.xml.bind.annotation.XmlNsForm;

This ensures that all the elements in the XML file are qualified - not just those relating to the fields in the ListBucketResult class.

Getters, Setters, and Field Names

I think the way you have named your Java fields is causing problems for your getters/setters, as they do not conform to the JavaBeans naming conventions.

Example: In your ListBucketResult, you have this:

private List<Contents> Contents;

public List<Contents> getContents() {
    return Contents;
}

public void setContents(List<Contents> contents) {
    Contents = contents;
}

The JavaBeans (and wider Java) naming convention is to use field names with camelcase - so, in this case contents. And then to match the getter and setter names accordingly - by capitalizing the first letter of the field: getContents, setContets.

For your XML, this means you also need to annotate the field name in the class, so that JAXB knows the Java field called contents maps to the XML element called Contents.

I choose to annotate the fields themselves - so I add an @XmlAccessorType annotation to the class, and then an @XmlElement annotation to the field.

I end up with this - ignoring all the other fields, for now:

import java.util.List;
import javax.xml.bind.annotation.XmlRootElement;
import javax.xml.bind.annotation.XmlElement;
import javax.xml.bind.annotation.XmlAttribute;
import javax.xml.bind.annotation.XmlAccessorType;
import javax.xml.bind.annotation.XmlAccessType;

@XmlRootElement(name = "ListBucketResult", namespace = "http://s3.amazonaws.com/doc/2006-03-01/")
@XmlAccessorType(XmlAccessType.FIELD)
public class ListBucketResult {

    @XmlElement(name="Name")
    private String name;

    @XmlElement(name="Contents")
    private List<Contents> contents;

    public List<Contents> getContents() {
        return contents;
    }

    public void setContents(List<Contents> contents) {
        this.contents = contents;
    }
}

Then, in the Contents class I have to do something similar - for example (again, ignoring all the fields except for key):

import javax.xml.bind.annotation.XmlElement;
import javax.xml.bind.annotation.XmlAccessType;
import javax.xml.bind.annotation.XmlAccessorType;

@XmlAccessorType(XmlAccessType.FIELD)
public class Contents {

    @XmlElement(name="Key")
    private String key;

    public String getKey() {
        return key;
    }
    public void setKey(String key) {
        this.key = key;
    }
}

If you repeat this process for the remaining fields in your classes, you will have fully populated Java objects.
